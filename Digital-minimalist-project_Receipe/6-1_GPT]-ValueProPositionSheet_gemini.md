# Value Proposition Sheet: 디지털 미니멀리스트 프로젝트 - 레시피 편 (Core User)

## 1. 개요

이 Value Proposition Sheet는 "디지털 미니멀리스트 프로젝트 - 레시피 편"의 핵심 고객 세그먼트인 "Core User (레시피 OS 빌더)"를 대상으로, 우리가 제공하는 제품/서비스가 고객의 고통을 어떻게 해결하고 이득을 창출하는지 분석합니다. 이는 초기 MVP 개발 및 시장 공략 전략 수립의 기반이 됩니다.

## 2. 고객 프로필 (Customer Profile)

### 2.1 고객 세그먼트 정의

**세그먼트: Core User (레시피 OS 빌더) - 디지털 미니멀리스트**

- **나이**: 20대 후반 ~ 40대 초반
- **직무/배경**: 주 3회 이상 집에서 요리하는 직장인, 주부, 또는 1인 가구
- **경험**:
    - 유튜브, 인스타그램, 블로그, 레시피 앱 등 **디지털 채널에서 레시피를 자주 찾음**
    - Notion, 카카오톡 서랍, 폴더 정리 등 **나름의 레시피 정리 시도를 이미 해봤으나 실패 경험**
    - **Notion 또는 유사 자동화 도구 사용에 익숙**하며, 새로운 자동화 툴에 대한 호기심과 실험 의지가 높음
    - 다이어트, 혈당 관리, 건강, 제철 식단 등 **특정 식단 관리에 꾸준히 관심**이 있는 편
- **특징**: "레시피를 하나의 OS처럼 정리하고 싶다"는 욕구가 뚜렷하며, "레시피 정리/추천"을 디지털 미니멀리즘 실험 프로젝트로 삼고 싶어 하는 얼리어답터 성향

### 2.2 고객 업무 (Customer Jobs)

고객이 달성하려는 일이나 해결하려는 문제를 나열합니다.

```
1. [Functional] 인스타/유튜브/블로그 등 여러 채널에 흩어진 레시피를 **한 군데에 자동 저장**하고 싶다.
2. [Functional] 저장된 레시피에 **재료/칼로리/제철 태그를 자동으로 정리**하여 검색 가능한 데이터베이스를 만들고 싶다.
3. [Emotional] 요리 직전에 "뭘 만들어야 하지?" 하는 **선택 과부하와 레시피 혼잡 스트레스**를 제거하고 싶다.
4. [Social] 정돈된 레시피 시스템을 통해 **요리 생활에 대한 자신감과 만족감**을 얻고 싶다.
5. [Functional] 다이어트/영양/제철/예산 등 **다양한 조건을 한 번에 고려**하여 최적의 레시피를 효율적으로 찾고 싶다.
```

### 2.3 고통 (Pains)

고객이 업무를 수행하면서 겪는 불편함, 리스크, 장애물을 나열합니다.

```
[Extreme Pains]
1. (레시피 분산 불편) 인스타, 유튜브, 블로그, 레시피 앱 등 **플랫폼마다 레시피가 분산**되어 있어 찾을 때마다 다시 검색하거나 북마크를 뒤져야 하는 스트레스 (AOS: 2.0)
2. (태그 자동화 부족) 저장해놓은 레시피에 **재료, 칼로리, 제철 등의 태그가 누락**되어 있어 검색이 어렵고 수동으로 정리해야 하는 번거로움 (AOS: 2.4)

[Moderate Pains]
3. (정리되지 않음) Notion 등에 레시피 DB를 만들어 본 적이 있지만, 출처, 재료, 칼로리, 사진 등이 뒤죽박죽이라 **검색이 안 되는 상태**로 관리 지옥을 경험함 (AOS: 3.0)
4. (선택 과부하) "뭘 만들어야 하지?" 할 때마다 **검색-스크랩-정리-재검색을 반복**해야 하며, 다이어트/영양/제철/예산 등 조건이 많을수록 선택 과부하가 폭발함 (TAM Pain)

[Light Pains]
5. (자동화에 대한 불안) "자동화가 제대로 안 되는 부분은 내가 또 고쳐야 하는 아닐까?" 하는 **초기 세팅 부담 및 불안감**
```

### 2.4 이득 (Gains)

고객이 원하는 결과, 혜택, 목표를 나열합니다.

```
[Required Gains]
1. (레시피 탐색 시간 단축) 레시피 탐색 시간을 **10분에서 10초**로 단축하여 요리 전 스트레스를 0으로 만들고 싶다.
2. (정돈된 요리 지식 시스템) 필요할 때 즉시 꺼내 쓸 수 있는 **정돈되고 검색 가능한 요리 지식 시스템**을 구축하고 싶다.

[Expected Gains]
3. (자동 태그 및 분류) 레시피 저장 시 **재료, 칼로리, 제철 등 필요한 태그가 자동으로 분류**되어 관리의 번거로움을 줄이고 싶다.
4. (통합된 정보) 다이어트, 영양, 제철, 예산 등 **조건이 많을수록 모든 정보가 한 번에 통합**되어 보이길 원한다.

[Desired Gains]
5. (요리 생활의 자신감) 정돈된 레시피 관리 시스템을 통해 **요리 과정에서의 효율성과 자신감**을 느끼고 싶다.
6. (새로운 자동화 경험) 새로운 자동화 도구(텔레그램 봇, 크롤러 등)를 통해 **레시피 관리가 혁신적으로 개선되는 경험**을 원한다.

[Unexpected Gains]
7. (레시피 OS 빌딩의 즐거움) 레시피 OS 구축 과정을 **자신의 사이드 프로젝트처럼 즐기며** 성취감을 느끼고 싶다.
```

## 3. 가치 맵 (Value Map)

### 3.1 제품과 서비스 (Products & Services)

우리가 제공하는 구체적인 제품, 서비스, 기능을 나열합니다.

```
[Core Products & Services (MVP)]
1. 멀티 플랫폼 레시피 크롤링 및 자동 수집 기능 (유튜브, 인스타그램, 블로그 등)
2. 표준 레시피 스키마 기반 데이터 정규화 및 저장
3. AI 기반 자동 태그링 (재료, 칼로리, 당류, 제철, 요리 종류 등)
4. 마크다운 / Notion 내보내기 및 연동 뷰어
5. 고급 검색 및 필터링 기능 (태그, 재료, 출처 기반)

[Supporting Services (향후 확장 예정)]
6. 개인화된 식단 추천 알고리즘 (선호 재료, 영양 목표 기반)
7. 자동 장보기 리스트 생성 및 연동 (주간 식단 플래너 기능)
8. 가족 구성원별 기호/알레르기 반영 기능
9. 커뮤니티 공유 및 큐레이션 기능
10. 제휴 마트/배달/헬스케어 앱 연동
```

### 3.2 고통 완화제 (Pain Relievers)

우리의 제품/서비스가 고객의 고통을 어떻게 완화하는지 구체적으로 설명합니다.

```
1. [Pain] 레시피 분산 불편 (AOS: 2.0)
   → [Reliever] **멀티 플랫폼 크롤링 및 자동 수집**으로 모든 레시피를 한 곳에 통합 저장
   - 유튜브, 인스타그램, 블로그 등 다양한 출처의 레시피를 자동으로 가져와 관리 부담 해소
   - 더 이상 여러 앱을 오가며 레시피를 찾아 헤매지 않아도 됨

2. [Pain] 태그 자동화 부족 (AOS: 2.4)
   → [Reliever] **AI 기반 자동 태그링**으로 재료, 칼로리, 제철 등 핵심 정보 자동 분류
   - 수동 태그 작업의 번거로움 제거 및 정확성 향상
   - "내가 좋아하는 인플루언서 + 제철재료 + 칼로리"까지 한 번에 보고 싶은 니즈 충족

3. [Pain] 정리되지 않음 (AOS: 3.0)
   → [Reliever] **표준 레시피 스키마 기반 데이터 정규화 및 마크다운/Notion 연동**으로 체계적인 DB 구축
   - 출처, 재료, 칼로리, 사진 등이 뒤죽박죽인 기존 DB의 문제 해결
   - Notion 사용자가 익숙한 형태로 데이터를 관리할 수 있도록 지원

4. [Pain] 선택 과부하 (TAM Pain)
   → [Reliever] **고급 검색 및 필터링 기능**으로 조건에 맞는 레시피를 10초 내에 탐색
   - 다이어트/영양/제철/예산 등 다양한 조건을 한 번에 적용하여 최적의 레시피를 쉽게 찾음
   - 검색-스크랩-정리-재검색의 반복 주기를 단축하여 피로도 감소

5. [Pain] 자동화에 대한 불안
   → [Reliever] **직관적인 사용자 인터페이스와 간편한 초기 설정**을 통해 자동화 진입 장벽 최소화
   - Notion 템플릿 제공 및 상세 가이드를 통해 사용자가 직접 쉽게 설정 가능
   - 문제가 발생할 경우를 대비한 피드백 채널 및 지원
```

### 3.3 이득 창출제 (Gain Creators)

우리의 제품/서비스가 고객이 원하는 이득을 어떻게 창출하는지 구체적으로 설명합니다.

```
[Required Gains 충족]
1. [Gain] 레시피 탐색 시간 10분 → 10초 단축
   → [Creator] **고급 검색 및 필터링 기능**과 **자동 태그링**으로 요리 직전 레시피 탐색 시간 혁신적으로 단축
   - 요리 전 스트레스 0, 정돈된 요리 지식 시스템 구축

2. [Gain] 정돈된 요리 지식 시스템 구축
   → [Creator] **표준 레시피 스키마 및 마크다운/Notion 연동**으로 개인화된 레시피 OS 제공
   - 필요할 때 즉시 꺼내 쓸 수 있는 체계적인 데이터베이스

[Expected Gains 충족]
3. [Gain] 재료, 칼로리, 제철 등 태그 자동 분류
   → [Creator] **AI 기반 자동 태그링**으로 정확하고 효율적인 레시피 정보 관리
   - 수동 작업 없이 필요한 정보에 쉽게 접근 가능

4. [Gain] 조건이 많을수록 모든 정보 통합
   → [Creator] **통합된 뷰와 고급 필터링 기능**으로 다이어트, 영양, 제철, 예산 등 복합 조건 충족
   - 여러 앱을 오갈 필요 없이 하나의 시스템에서 모든 정보 확인

[Desired Gains 충족]
5. [Gain] 요리 과정의 효율성과 자신감
   → [Creator] **간편하고 자동화된 레시피 관리**로 요리 과정의 전반적인 만족도 향상
   - 매번 검색하거나 정리하는 부담 없이 요리에 집중

6. [Gain] 새로운 자동화 경험
   → [Creator] **최첨단 크롤링 및 AI 기술**을 활용한 혁신적인 레시피 관리 경험 제공
   - 디지털 미니멀리즘을 실현하는 선도적인 서비스 경험

[Unexpected Gains 충족]
7. [Gain] 레시피 OS 빌딩의 즐거움
   → [Creator] **사용자 맞춤형 설정 및 확장성 높은 Notion 연동**으로 나만의 레시피 OS를 구축하는 재미 제공
   - 개발자가 된 듯한 성취감과 주도적인 데이터 관리 경험
```

## 4. 적합성(Fit) 평가

### 4.1 Problem-Solution Fit 평가

**Pain-Reliever Fit:** 우리의 Pain Reliever는 Core User의 가장 중요한 Pain인 **레시피 분산, 태그 자동화 부족, 정리되지 않은 레시피 DB 문제를 효과적으로 해결**합니다. 특히, 멀티 플랫폼 크롤링과 AI 기반 자동 태그링은 고객의 핵심 고통을 직접적으로 완화하여 높은 만족도를 제공할 것으로 예상됩니다.

**Gain-Creator Fit:** 우리의 Gain Creator는 Core User가 가장 원하는 Gain인 **레시피 탐색 시간 단축, 정돈된 요리 지식 시스템 구축, 자동 태그 및 통합된 정보 제공**을 충족시킵니다. 이를 통해 고객은 요리 전 스트레스 없는 효율적인 요리 생활을 경험하고, 새로운 자동화 기술을 통해 혁신적인 가치를 얻을 수 있을 것입니다.

### 4.2 우선순위 매트릭스

"Core User (레시피 OS 빌더)"는 **High AOS (고통 강도 높음)와 High DOS (시장성 높음)**를 가진 Q1 핵심 혁신 기회 그룹으로, MVP에 반드시 포함되어야 할 기능들을 중심으로 우선순위를 매겼습니다.

```
   높음 │ Priority 1 (MVP 핵심)    Priority 2 (Roadmap 중기)
중  │ (레시피 자동 수집, AI 태그링) (식단 추천, 장보기 리스트)
요  │
도  │
   │
   낮음 │ Priority 3 (Quick Win)   Priority 4 (장기/제외)
        │ (간편 공유, 템플릿 제공)  (제휴 연동, 언어 확장)
        └─────────────────────────────
           낮음          높음
              제공 난이도
```

**Priority 1 (Must Have): MVP에 반드시 포함**
- 멀티 플랫폼 레시피 크롤링 및 자동 수집
- AI 기반 자동 태그링 (재료, 칼로리, 당류, 제철, 요리 종류)
- 표준 레시피 스키마 기반 데이터 정규화 및 마크다운/Notion 내보내기 뷰어
- 고급 검색 및 필터링 기능

**Priority 2 (Nice to Have): 로드맵 상 중기 목표**
- 개인화된 식단 추천 알고리즘
- 자동 장보기 리스트 생성 및 연동
- 가족 구성원별 기호/알레르기 반영 기능

**Priority 3 (Good to Have): 빠른 승리 (Quick Win)**
- 간편한 레시피 공유 및 큐레이션 기능
- Notion 템플릿 및 사용 가이드 제공 강화

**Priority 4 (Not Critical): 나중에 또는 제외**
- 제휴 마트/배달/헬스케어 앱 연동
- 언어/지역 확장

## 5. Value Proposition 한 문장 요약

```
우리의 "디지털 레시피 OS 서비스"는
"레시피 OS 빌더 (Core User)"가
인스타/유튜브/블로그 등 여러 채널에 흩어진 레시피를 한 곳에 모으고, 재료/칼로리/제철 태그를 자동으로 정리하여 검색 가능한 데이터베이스를 만들 때
플랫폼 분산, 태그 누락, 정리되지 않은 DB로 인한 스트레스와 선택 과부하를 제거하고
레시피 탐색 시간을 10분에서 10초로 단축하며, 정돈된 요리 지식 시스템을 통해 요리 과정의 효율성과 자신감을 제공합니다.
```

## 6. 검증 및 반복 (Validation & Iteration)

Value Proposition은 가설이므로 지속적인 검증이 필요합니다. Core User를 대상으로 한 MVP 실험과 고객 인터뷰를 통해 Pain-Reliever Fit 및 Gain-Creator Fit을 지속적으로 검증하고 개선할 예정입니다.

### 6.1 검증 방법

- **고객 인터뷰**: JTBD 인터뷰를 통해 Core User의 Pain과 Gain의 실재성을 지속적으로 확인 (10-20회)
- **MVP 실험**: 초기 MVP를 베타 유저 (SOM 대상 20-50명)에게 제공하여 실제 사용 및 만족도 확인
- **설문조사**: 기능별 우선순위 및 지불 의향 확인

### 6.2 반복 개선

- **매 분기**: 고객 피드백 기반 Value Map 업데이트 및 기능 개선
- **매 6개월**: Customer Profile 재검증 (시장 변화 및 Adjacent User 확장 고려)
- **매년**: 전체 Value Proposition 재정의

## 9. 부록: 용어 정의

| 용어 | 정의 |
|------|------|
| **Value Proposition** | 제품/서비스가 고객에게 제공하는 가치에 대한 약속 |
| **Customer Profile** | 고객의 특성, 업무, 고통, 이득을 체계적으로 정리한 것 |
| **Value Map** | 제품/서비스가 제공하는 가치 요소를 체계적으로 정리한 것 |
| **Fit** | 고객이 원하는 것과 우리가 제공하는 것 간의 일치 정도 |
| **Jobs To Be Done (JTBD)** | 고객이 특정 상황에서 달성하려는 진보 또는 해결하려는 문제 |
| **Pain Relievers** | 고객의 고통을 완화하거나 제거하는 제품/서비스의 속성 |
| **Gain Creators** | 고객이 원하는 이득을 창출하는 제품/서비스의 속성 |
| **MVP (Minimum Viable Product)** | 최소 기능 제품 - 가설 검증을 위한 최소한의 기능을 갖춘 제품 |
