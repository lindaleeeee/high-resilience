# Value Proposition Sheet: 디지털 미니멀리스트 프로젝트 - 레시피 편 (GPT5.1 버전)

## 1. 개요

이 문서는 “디지털 미니멀리스트 프로젝트 - 레시피 편”을 기획·실행하려는 **예비 창업자**를 위한 Value Proposition Sheet입니다. 제공된 데이터(TAM-SAM-SOM, 페르소나 분석, AOS×DOS 고통·만족도 평가, JTBD 인터뷰)를 기반으로, **Core User(레시피 OS 빌더)** 세그먼트에서 문제-해결-가치 정합성을 명확히 정리하여 초기 MVP 방향성과 시장 접근 전략을 빠르게 설계할 수 있도록 돕습니다.

## 2. 고객 프로필 (Customer Profile)

### 2.1 고객 세그먼트 정의

**세그먼트: Core User (레시피 OS 빌더) – 디지털 미니멀리스트**

- **나이**: 20대 후반 ~ 40대 초반
- **생활 패턴**: 주 3회 이상 집에서 요리하는 직장인/주부/1인 가구
- **행동 특징**
  - 유튜브·인스타·블로그·레시피 앱 등 **2개 이상 디지털 채널에서 레시피를 탐색**
  - Notion, 카카오톡 서랍, 폴더 등으로 **레시피 정리 시도를 이미 해봤으나 유지 실패 경험**
  - 새로운 자동화 도구(텔레그램 봇, 크롤러, 구글 시트 등)에 대한 **호기심과 실험 의지**가 높음
  - 다이어트, 혈당, 제철, 예산 등 **식단 조건 관리에 꾸준한 관심**
- **욕구**: “레시피를 수집용이 아니라 **내 삶의 운영체제(OS)**처럼 관리하고 싶다”는 의지가 강하며, 레시피 정리·추천을 **디지털 미니멀리즘 실험 프로젝트**로 즐김

### 2.2 고객 업무 (Customer Jobs)

```
1. [Functional] 인스타/유튜브/블로그 등 여러 채널에 흩어진 레시피를 한 곳에 자동 저장하고 싶다.
2. [Functional] 저장된 레시피에 재료/칼로리/제철 태그를 자동으로 정리해 검색 가능한 DB를 만들고 싶다.
3. [Emotional] 요리 직전에 “뭘 만들어야 하지?” 하는 선택 과부하와 레시피 혼잡 스트레스를 없애고 싶다.
4. [Social] 정돈된 레시피 시스템을 통해 요리 생활에서의 자신감과 만족감을 얻고 싶다.
5. [Functional] 다이어트/영양/제철/예산 등 복합 조건을 한 번에 고려해 최적의 레시피를 효율적으로 찾고 싶다.
```

### 2.3 고통 (Pains)

```
[Extreme Pains]
1. (레시피 분산 불편, AOS 2.0) 플랫폼마다 레시피가 흩어져 있어 찾으려면 다시 검색하거나 북마크를 뒤져야 함
2. (태그 자동화 부족, AOS 2.4) 재료·칼로리·제철 태그가 누락돼 검색이 어렵고 수동 정리가 필요함

[Moderate Pains]
3. (정리되지 않음, AOS 3.0) Notion DB를 만들어도 출처·재료·사진 등이 뒤섞여 검색이 되지 않는 관리 지옥
4. (선택 과부하) “뭘 만들어야 하지?” 할 때마다 검색-스크랩-정리-재검색을 반복하며 조건이 많을수록 과부하

[Light Pains]
5. (자동화 불안) “자동화가 제대로 안 되면 내가 또 손봐야 하는 것 아닐까?”라는 초기 세팅 부담과 불안감
```

### 2.4 이득 (Gains)

```
[Required Gains]
1. (레시피 탐색 시간 단축) 레시피 탐색 시간을 10분 → 10초로 줄여 요리 전 스트레스를 0으로 만들고 싶다.
2. (정돈된 요리 지식 시스템) 필요할 때 즉시 꺼내 쓸 수 있는 정돈되고 검색 가능한 레시피 OS를 구축하고 싶다.

[Expected Gains]
3. (자동 태그 분류) 레시피 저장 시 재료·칼로리·제철 등 필요한 태그가 자동 분류되길 원한다.
4. (통합 정보) 다이어트/영양/제철/예산 등 조건이 많을수록 모든 정보가 한 번에 통합되어 보이길 원한다.

[Desired Gains]
5. (요리 자신감) 정돈된 시스템으로 요리 과정의 효율성과 자신감을 느끼고 싶다.
6. (새로운 자동화 경험) 텔레그램 봇·크롤러 등 새로운 자동화 도구로 레시피 관리가 혁신적으로 개선되길 원한다.

[Unexpected Gains]
7. (레시피 OS 빌딩의 즐거움) 레시피 OS 구축 과정을 사이드 프로젝트처럼 즐기며 성취감을 느끼고 싶다.
```

## 3. 가치 맵 (Value Map)

### 3.1 제품과 서비스 (Products & Services)

```
[Core Products & Services (MVP)]
1. 멀티 플랫폼 레시피 크롤링/자동 수집 (유튜브, 인스타그램, 블로그 등)
2. 표준 레시피 스키마 기반 데이터 정규화 및 저장
3. AI 자동 태그링 (재료, 칼로리, 당류, 제철, 요리 종류 등)
4. 마크다운/Notion 내보내기 및 연동 뷰어
5. 고급 검색·필터링 (태그, 재료, 출처 기반)

[Supporting Services (향후 확장)]
6. 개인화 식단 추천 알고리즘 (선호 재료·영양 목표 기반)
7. 자동 장보기 리스트 생성 및 주간 식단 플래너
8. 가족 구성원별 기호/알레르기 반영 기능
9. 커뮤니티 공유·큐레이션 기능
10. 제휴 마트/배달/헬스케어 앱 연동
```

### 3.2 고통 완화제 (Pain Relievers)

```
1. 레시피 분산 불편 → 멀티 플랫폼 크롤링/자동 수집으로 모든 레시피를 한 곳에 통합 저장
2. 태그 자동화 부족 → AI 자동 태그링으로 재료·칼로리·제철 등 핵심 정보 즉시 분류
3. 정리되지 않음 → 표준 스키마·마크다운/Notion 연동으로 체계적인 DB 구축
4. 선택 과부하 → 고급 검색/필터링으로 원하는 조건의 레시피를 10초 내 탐색
5. 자동화 불안 → 직관적 UI와 간편한 초기 설정, 템플릿·가이드·지원 채널 제공
```

### 3.3 이득 창출제 (Gain Creators)

```
[Required]
1. 레시피 탐색 10분 → 10초: 고급 검색·필터링 + 자동 태그링
2. 정돈된 레시피 OS: 표준 스키마 + Notion/마크다운 연동

[Expected]
3. 자동 태그 분류: AI 태그링으로 정확·효율적 관리
4. 조건 통합: 통합 뷰 + 고급 필터링으로 복합 조건 충족

[Desired]
5. 요리 자신감: 간편·자동화된 레시피 관리로 전반 만족도 향상
6. 새로운 자동화 경험: 최첨단 크롤링/AI 기술로 혁신적 사용자 경험

[Unexpected]
7. 레시피 OS 빌딩의 즐거움: 사용자 맞춤 설정·확장성 높은 Notion 연동
```

## 4. 적합성(Fit) 평가

### 4.1 Problem-Solution Fit

- Pain-Reliever Fit: 멀티 플랫폼 크롤링, AI 태그링, 표준 스키마는 Core User의 핵심 Pain(레시피 분산/정리 불가/태그 부재)을 직접 해결
- Gain-Creator Fit: 탐색 시간 단축, 정돈된 OS, 자동 태그·통합 정보 제공을 통해 효율적 요리 경험과 디지털 미니멀리즘 가치를 제공

### 4.2 우선순위 매트릭스

```
  높음 │ Priority 1 (MVP 핵심)    Priority 2 (로드맵 중기)
중요 │ (크롤링, AI 태그, 정규화,   (식단 추천, 장보기 리스트,
도   │  고급 검색)                 가족 기호 반영)
      │
  낮음 │ Priority 3 (Quick Win)     Priority 4 (장기/제외)
      │ (템플릿/가이드, 공유 기능)  (제휴 연동, 언어/지역 확장)
      └─────────────────────────────
         낮음             높음
              제공 난이도
```

- **Priority 1**: 멀티 플랫폼 크롤링, AI 태그링, 표준 스키마 정규화, 고급 검색/필터
- **Priority 2**: 개인화 식단 추천, 자동 장보기 리스트, 가족 맞춤 기능
- **Priority 3**: Notion 템플릿/가이드 강화, 간편 공유/큐레이션
- **Priority 4**: 제휴 연동, 언어·지역 확장

## 5. Value Proposition 한 문장 요약

```
우리의 “디지털 레시피 OS 서비스”는
Core User(레시피 OS 빌더)가 다양한 채널에 흩어진 레시피를 한 곳에 모으고,
재료/칼로리/제철 태그를 자동 정리해 검색 가능한 데이터베이스를 만들 때,
플랫폼 분산, 태그 누락, 정리되지 않은 DB로 인한 스트레스와 선택 과부하를 제거하고,
레시피 탐색 시간을 10분에서 10초로 단축하며 정돈된 요리 지식 시스템을 제공한다.
그 방법은 멀티 플랫폼 자동 크롤링, AI 태그링, Notion/마크다운 연동 기반의 개인화 레시피 OS 구축이다.
```
