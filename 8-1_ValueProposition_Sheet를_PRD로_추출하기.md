# Value Proposition Sheet 작성 방법론

## 1. 개요

Value Proposition Sheet(가치 제안서)는 고객이 원하는 것과 우리가 제공하는 솔루션 간의 적합성(Fit)을 체계적으로 분석하고 정리하는 도구입니다. 

### 1.1 Value Proposition Canvas의 구조

Value Proposition Canvas는 크게 두 부분으로 구성됩니다:
- **고객 프로필(Customer Profile)**: 고객에 대한 이해
- **가치 맵(Value Map)**: 우리가 제공하는 가치

```
┌─────────────────────────────────────────┐
│        고객 프로필                       │
│  (Customer Profile)                    │
│                                         │
│  ① 고객 업무 (Customer Jobs)           │
│  ② 고통 (Pains)                        │
│  ③ 이득 (Gains)                        │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│        가치 맵                           │
│  (Value Map)                           │
│                                         │
│  ① 제품과 서비스 (Products & Services) │
│  ② 고통 완화제 (Pain Relievers)        │
│  ③ 이득 창출제 (Gain Creators)         │
└─────────────────────────────────────────┘
```

## 2. 고객 프로필 (Customer Profile) 작성 방법

### 2.1 고객 세그먼트 정의

가장 먼저, 분석하고자 하는 **고객 세그먼트**를 명확히 정의합니다.

**작성 가이드:**
- 페르소나 스펙트럼 (Core, Adjacent, Extreme, Non-user) 중 어디에 해당하는가?
- 직무, 산업, 조직 규모, 경험 수준 등을 구체적으로 명시
- 한 장의 Value Proposition Sheet는 하나의 세그먼트에 집중

**예시:**
```
세그먼트: 코어 - 초보 예비창업자
- 나이: 30-40대
- 직무/배경: 직장인 또는 프리랜서로 첫 창업 준비 중
- 조직 규모: 1-3인 (본인 포함)
- 경험: 사업 경험 1년 미만, 사업계획 수립 경험 없음
```

### 2.2 고객 업무 (Customer Jobs)

고객이 **달성하려는 일**이나 **해결하려는 문제**를 나열합니다.

**업무의 3가지 유형:**

| 유형 | 설명 | 예시 |
|------|------|------|
| **Functional Jobs** | 특정 과제나 문제 해결 | "시장 조사를 완료해야 함", "사업계획서를 작성해야 함" |
| **Social Jobs** | 사회적 지위, 이미지 관련 | "전문가처럼 보이고 싶음", "투자자에게 신뢰를 얻고 싶음" |
| **Emotional Jobs** | 감정적 상태 관련 | "실패에 대한 불안을 줄이고 싶음", "자신감을 갖고 싶음" |

**작성 가이드:**
- JTBD (Jobs To Be Done) 프레임워크 활용
- "When [상황], I want to [하고 싶은 일], so I can [원하는 결과]" 형식으로 작성
- 우선순위가 높은 5-10개의 업무를 선정

**예시:**
```
1. [Functional] 월말 경영회의 전에 지표를 한 번에 모아 신뢰성 있는 리포트를 제출하고자 함
2. [Functional] 비용 절감을 위한 구체적인 실행 방안을 도출하고자 함
3. [Social] 투자자 앞에서 전문적이고 설득력 있는 사업 계획을 제시하고자 함
4. [Emotional] 창업 실패에 대한 불안감을 줄이고 자신감을 갖고 싶음
```

### 2.3 고통 (Pains)

고객이 업무를 수행하면서 겪는 **불편함, 리스크, 장애물**을 나열합니다.

**고통의 3가지 레벨:**

| 레벨 | 설명 | 예시 |
|------|------|------|
| **Extreme** | 업무 완수 자체를 불가능하게 만드는 요인 | "시장 조사 방법을 전혀 모름", "무엇부터 해야 할지 모름" |
| **Moderate** | 업무를 어렵게 만드는 요인 | "데이터 수집에 시간이 너무 많이 걸림", "전문 용어가 어려움" |
| **Light** | 경미한 불편 | "양식이 복잡함", "예시가 부족함" |

**작성 가이드:**
- 현재 고객이 사용하는 대체 솔루션의 문제점 포함
- "너무 ~하다", "~이/가 부족하다", "~을/를 모른다" 형식으로 작성
- JTBD 인터뷰의 4 Forces 중 Push (현상 불만), Anxiety (불안) 활용

**예시:**
```
[Extreme Pains]
1. 무엇이 "좋은 사업계획"인지에 대한 명확한 기준이 없어 방향을 잡을 수 없음
2. 실패 요인을 사전에 파악할 방법이 없어 수억 원의 시행착오 비용이 두려움

[Moderate Pains]
3. 시장 조사, 경쟁 분석, 재무 계획 등 전문 지식이 필요한 작업에 시간이 너무 많이 소요됨
4. 기존 컨설팅은 비용이 수백만 원으로 너무 높아 접근하기 어려움

[Light Pains]
5. 인터넷에 있는 사업계획서 템플릿들이 너무 일반적이고 우리 상황에 맞지 않음
```

### 2.4 이득 (Gains)

고객이 **원하는 결과, 혜택, 목표**를 나열합니다.

**이득의 4가지 유형:**

| 유형 | 설명 | 예시 |
|------|------|------|
| **Required Gains** | 반드시 필요한 기본 결과 | "사업계획서가 완성되어야 함" |
| **Expected Gains** | 기대하는 결과 | "시장 조사가 체계적이고 설득력 있어야 함" |
| **Desired Gains** | 바라는 추가 혜택 | "시간을 절약하고 싶음", "전문가 수준의 품질" |
| **Unexpected Gains** | 예상치 못한 놀라운 혜택 | "창업가 커뮤니티 접근", "지속적인 멘토링" |

**작성 가이드:**
- JTBD 인터뷰의 Desired Outcome (측정 가능한 결과) 활용
- "~이/가 되어야 함", "~을/를 원함", "~하면 좋겠음" 형식으로 작성
- 정량적 목표 포함 시 더욱 효과적

**예시:**
```
[Required Gains]
1. 투자자/은행에 제출 가능한 형태의 사업계획서가 완성되어야 함
2. 시장 규모와 목표 고객이 명확히 정의되어야 함

[Expected Gains]
3. 예상 실패 요인을 사전에 파악하고 대응 방안을 마련할 수 있어야 함
4. 월 현금흐름 목표를 달성하기 위한 현실적인 로드맵이 있어야 함

[Desired Gains]
5. 사업계획 작성 시간을 80% 이상 단축하고 싶음 (예: 4주 → 1주 이내)
6. 전문 컨설턴트 수준의 분석 품질을 얻고 싶음

[Unexpected Gains]
7. 유사한 창업가들과 네트워킹할 수 있는 커뮤니티에 접근하고 싶음
8. 사업 운영 중에도 지속적으로 자문을 받을 수 있으면 좋겠음
```

## 3. 가치 맵 (Value Map) 작성 방법

### 3.1 제품과 서비스 (Products & Services)

우리가 제공하는 **구체적인 제품, 서비스, 기능**을 나열합니다.

**작성 가이드:**
- 고객에게 보이는 모든 요소 포함 (물리적 제품, 디지털 서비스, 컨설팅, 콘텐츠 등)
- 기능 중심이 아닌, 고객 관점에서 인식되는 "제공물" 중심으로 작성
- MVP부터 향후 확장 계획까지 포함 가능

**예시:**
```
[Core Products & Services]
1. 3단계 종합 진단 서비스
   - 1단계: 관점(Mindset) 진단
   - 2단계: 실행(Execution) 진단  
   - 3단계: 창업가(Founder) 진단

2. 개인화된 진단 리포트 및 처방전

3. "생존형 비즈니스 계획서" 템플릿 및 가이드

4. "창업 50계명" 체크리스트

[Supporting Services]
5. 1:1 전문가 상담 (1-2회)

6. 온라인 워크숍 및 그룹 코칭

7. 실행 과제 및 피드백 사이클

[Content & Tools]
8. 사업계획서 작성 템플릿 라이브러리

9. 시장 분석 도구 (TAM-SAM-SOM, Porter's 5 Forces 등)

10. 교육 콘텐츠 (비디오, 아티클, 케이스 스터디)
```

### 3.2 고통 완화제 (Pain Relievers)

우리의 제품/서비스가 **고객의 고통을 어떻게 완화하는지** 구체적으로 설명합니다.

**작성 가이드:**
- 앞서 정의한 고객의 Pains와 **1:1 매칭**되도록 작성
- "어떻게(How)" 완화하는지 메커니즘 설명
- 정량적 지표 포함 시 더욱 효과적 (시간 절감, 비용 절감 등)

**매칭 구조 예시:**

| 고객의 고통 (Pain) | 우리의 완화제 (Pain Reliever) |
|-------------------|------------------------------|
| 무엇이 "좋은 사업계획"인지 기준이 없음 | "생존형 비즈니스 계획서" 프레임워크 제공으로 명확한 기준 제시 |
| 실패 요인 파악 방법이 없음 | "창업 50계명" 기반 진단으로 예측 가능한 실패 요인 사전 발견 |
| 전문 지식이 필요한 작업에 시간이 많이 소요됨 | 템플릿, 도구, 가이드를 통해 작업 시간 80% 단축 (4주 → 1주) |
| 기존 컨설팅은 비용이 너무 높음 | 온라인 자동화 + 필요 시에만 전문가 개입하는 하이브리드 모델로 비용 1/10 수준으로 절감 |
| 일반적인 템플릿은 우리 상황에 맞지 않음 | 3단계 진단을 통한 개인화된 맞춤형 리포트 및 처방 제공 |

**작성 예시:**
```
1. [Pain] 무엇이 "좋은 사업계획"인지 기준이 없음
   → [Reliever] "생존형 비즈니스 계획서" 프레임워크로 명확한 기준 제시
   - 비전이 아닌 "월 현금흐름 목표" 중심
   - 단계별 체크리스트 제공
   - 실제 사례 기반 벤치마크 제공

2. [Pain] 실패 요인을 사전에 파악할 방법이 없음
   → [Reliever] "창업 50계명" 기반 실행 진단으로 예측 가능한 실패 요인 조기 발견
   - 각 계명별 위험도 점수화
   - "실패 위험 Top 3" 우선순위 제시
   - 구체적인 대응 방안 제공

3. [Pain] 전문 지식 필요 작업에 시간 과다 소요
   → [Reliever] 템플릿 + 도구 + 가이드로 작업 시간 80% 단축
   - 즉시 사용 가능한 템플릿 라이브러리
   - 단계별 작성 가이드
   - 자동화된 분석 도구 (TAM-SAM-SOM 계산기 등)

4. [Pain] 기존 컨설팅 비용이 너무 높음
   → [Reliever] 온라인 자동화 + 선택적 전문가 개입으로 비용 1/10 절감
   - 자가 진단 및 리포트는 자동화
   - 필요 시에만 1:1 전문가 상담 제공
   - 구독 모델로 월 단위 소액 결제 가능
```

### 3.3 이득 창출제 (Gain Creators)

우리의 제품/서비스가 **고객이 원하는 이득을 어떻게 창출하는지** 구체적으로 설명합니다.

**작성 가이드:**
- 앞서 정의한 고객의 Gains와 **1:1 매칭**되도록 작성
- Required/Expected Gains는 반드시 충족해야 함
- Desired/Unexpected Gains 충족 시 강력한 차별화 요소

**매칭 구조 예시:**

| 고객의 이득 (Gain) | 우리의 창출제 (Gain Creator) |
|-------------------|------------------------------|
| 투자자에게 제출 가능한 사업계획서 완성 | 투자 유치에 최적화된 "생존형 비즈니스 계획서" 템플릿 및 사례 제공 |
| 예상 실패 요인 파악 및 대응 방안 마련 | "창업 50계명" 진단을 통한 실패 요인 Top 3 + 구체적 대응 로드맵 제공 |
| 사업계획 작성 시간 80% 단축 | 자동화 도구 + 템플릿으로 4주 → 1주 이내 완성 가능 |
| 전문 컨설턴트 수준의 품질 | 데이터 기반 진단 + 전문가 검토로 전문성 보장 |
| 창업가 커뮤니티 접근 | 그룹 워크숍 및 온라인 커뮤니티 제공 |

**작성 예시:**
```
[Required Gains 충족]
1. [Gain] 투자자/은행 제출용 사업계획서 완성
   → [Creator] "생존형 비즈니스 계획서" 템플릿으로 투자자가 원하는 형식 제공
   - 실제 투자 유치 성공 사례 기반
   - 재무 계획 템플릿 포함
   - 투자자 프레젠테이션 슬라이드 자동 생성

2. [Gain] 시장 규모와 목표 고객 명확화
   → [Creator] TAM-SAM-SOM 분석 도구 + 페르소나 스펙트럼 템플릿
   - 단계별 가이드로 쉽게 완성
   - 데이터 소스 제공
   - 시각화 자동 생성

[Expected Gains 충족]
3. [Gain] 예상 실패 요인 파악 및 대응
   → [Creator] "창업 50계명" 진단으로 실패 위험 Top 3 + 대응 로드맵
   - 각 위험 요인별 구체적인 Action Item
   - 우선순위 및 타임라인 제시
   - 진행 상황 추적 도구

4. [Gain] 월 현금흐름 목표 달성 로드맵
   → [Creator] 생존 중심 재무 계획 + 단계별 마일스톤
   - 손익분기점(BEP) 달성 로드맵
   - 월별 목표 및 KPI 설정
   - 실행 과제 분해 (Breakdown)

[Desired Gains 충족]
5. [Gain] 사업계획 작성 시간 80% 단축
   → [Creator] 자동화 도구 + 템플릿으로 4주 → 1주 완성
   - 1일차: 진단 및 분석
   - 3일차: 템플릿 기반 초안 작성
   - 7일차: 전문가 피드백 반영 후 완성

6. [Gain] 전문 컨설턴트 수준 품질
   → [Creator] 데이터 기반 자동 진단 + 전문가 최종 검토
   - 50+ 창업 실패 사례 데이터 기반
   - 업계 전문가의 피드백
   - 버전 관리 및 지속적 개선

[Unexpected Gains 충족]
7. [Gain] 창업가 커뮤니티 접근
   → [Creator] 온라인 커뮤니티 + 그룹 워크숍 정기 개최
   - 동료 창업가와의 네트워킹
   - 상호 피드백 및 학습
   - 월 1회 그룹 코칭 세션

8. [Gain] 사업 운영 중 지속 자문
   → [Creator] 구독 모델로 지속적인 업데이트 및 자문 제공
   - 분기별 재진단 서비스
   - 실시간 Q&A 채널
   - 신규 도구 및 콘텐츠 지속 제공
```

---

## 4. 적합성(Fit) 평가

고객 프로필과 가치 맵 간의 적합성을 평가합니다.

### 4.1 Problem-Solution Fit 평가

| 평가 기준 | 질문 | 평가 방법 |
|-----------|------|-----------|
| **Pain-Reliever Fit** | 우리의 Pain Reliever가 고객의 가장 중요한 Pain을 해결하는가? | - 각 Pain별 중요도(1-5) 평가<br>- 각 Reliever별 효과성(1-5) 평가<br>- 중요도 × 효과성 점수 산출 |
| **Gain-Creator Fit** | 우리의 Gain Creator가 고객이 가장 원하는 Gain을 창출하는가? | - 각 Gain별 중요도(1-5) 평가<br>- 각 Creator별 효과성(1-5) 평가<br>- 중요도 × 효과성 점수 산출 |

### 4.2 우선순위 매트릭스

```
   높음 │ Priority 1       Priority 2
중  │ (Must Have)      (Nice to Have)
요  │
도  │
   │
   낮음 │ Priority 3       Priority 4
        │ (Good to Have)   (Not Critical)
        └─────────────────────────────
           낮음          높음
              제공 난이도
```

**Priority 1 (Must Have)**: 중요도 높음 + 난이도 낮음 → MVP에 반드시 포함
**Priority 2 (Nice to Have)**: 중요도 높음 + 난이도 높음 → 로드맵 상 중기 목표
**Priority 3 (Good to Have)**: 중요도 낮음 + 난이도 낮음 → 빠른 승리(Quick Win)
**Priority 4 (Not Critical)**: 중요도 낮음 + 난이도 높음 → 나중에 또는 제외

## 5. Value Proposition 한 문장 요약

분석 결과를 바탕으로 Value Proposition을 한 문장으로 요약합니다.

**템플릿:**
```
우리의 [제품/서비스]는
[목표 고객 세그먼트]가
[핵심 고객 업무]를 수행할 때
[주요 고통]을 해결하고
[주요 이득]을 제공합니다.
그 방법은 [핵심 차별점]입니다.
```

**예시:**
```
우리의 "3단계 종합 진단 온라인 컨설팅"은
처음으로 사업을 준비하는 초보 예비창업자가
생존 가능한 사업계획을 수립할 때
막연한 불안과 수억 원의 예측 가능한 실패 비용을 제거하고
1주일 내에 투자 가능한 수준의 사업계획서와 실행 로드맵을 완성하도록 돕습니다.
그 방법은 "실패하지 않는 법" 중심의 생존형 프레임워크와 데이터 기반 자동 진단입니다.
```

## 6. 검증 및 반복 (Validation & Iteration)

Value Proposition은 가설이므로 지속적인 검증이 필요합니다.

### 6.1 검증 방법

| 방법 | 목적 | 실행 |
|------|------|------|
| **고객 인터뷰** | Pain과 Gain의 실재성 확인 | JTBD 인터뷰 10-20회 |
| **설문조사** | 우선순위 및 지불 의향 확인 | 타깃 고객 50-100명 대상 |
| **랜딩 페이지 테스트** | 메시지 반응도 측정 | A/B 테스트로 전환율 비교 |
| **MVP 실험** | 실제 사용 및 만족도 확인 | 베타 유저 20-50명 대상 |
| **가격 테스트** | 가격 민감도 및 지불 의향 확인 | Van Westendorp PSM 분석 |

## 부록: 용어 정의

| 용어 | 정의 |
|------|------|
| **Value Proposition** | 제품/서비스가 고객에게 제공하는 가치에 대한 약속 |
| **Customer Profile** | 고객의 특성, 업무, 고통, 이득을 체계적으로 정리한 것 |
| **Value Map** | 제품/서비스가 제공하는 가치 요소를 체계적으로 정리한 것 |
| **Fit** | 고객이 원하는 것과 우리가 제공하는 것 간의 일치 정도 |
| **Jobs To Be Done (JTBD)** | 고객이 특정 상황에서 달성하려는 진보 또는 해결하려는 문제 |
| **Pain Relievers** | 고객의 고통을 완화하거나 제거하는 제품/서비스의 속성 |
| **Gain Creators** | 고객이 원하는 이득을 창출하는 제품/서비스의 속성 |
| **MVP (Minimum Viable Product)** | 최소 기능 제품 - 가설 검증을 위한 최소한의 기능을 갖춘 제품 |
