    # ✅ 1. “전환 여정” Mermaid 다이어그램 (JTBD 4 Forces 기반)

    ```mermaid

        AA[상황적 어려움<br>레시피가 여기저기 흩어져 있음<br>요리 직전마다 다시 검색함]
            --> |"상황 불만 (Push)"|A[현재의 나<br>레시피 혼잡 스트레스]
    ```

    # ✅ 2. JTBD 인터뷰 요약 카드 (디지털 미니멀리즘 *레시피OS* 버전)
    
    नी
    
    아래는 “예시 1번 카드 구조 그대로” **레시피 자동정리 서비스 사용자 JTBD**를 4 Forces 포함해 재작성한 JTBD Cards야.

    # 📌 **카드 1 — Core User (레시피 OS 빌더)**
    
    *(현재 MVP 즉시 도입 가능층 / SOM Target)*
    
    | 구분 | 내용 |
    | --- | --- |
    | **Persona / Segment** | **Core – 레시피 OS 빌더 (디지털 미니멀리스트)**Notion·시트·AI를 이미 쓰지만 “정리 단순화”가 필요함 |
    | **Situation (When …)** | 인스타·유튜브·블로그 레시피를 저장해놓았지만 **찾을 때마다 다시 검색하거나 북마크를 뒤지는 상황** |
    | **Job Statement** | 레시피를 **한 군데에 자동 저장**하고, **재료/칼로리/제철 태그까지 자동으로 정리된 레시피 데이터베이스**를 갖고 싶다 |
    | **Desired Outcome** | • 레시피 탐색 시간 **10분 → 10초**로 단축• 필요할 때 즉시 꺼내쓸 수 있는 **정돈된 요리 지식 시스템** 구축 |
    | **4 Forces** | **Push:** 저장해놓은 레시피를 전혀 못 찾는 스트레스**Pull:** 자동 수집 + 자동 태그 + 검색 가능한 OS에 대한 기대감**Habit:** 인스타 저장·유튜브 좋아요·카톡 레시피 사진에 계속 매달림**Anxiety:** “자동화가 제대로 안 되는 부분은 내가 또 고쳐야 하는 건 아닐까?” |
    | **Current Solutions** | Notion 수동 복붙, 인스타 저장함, 네이버 북마크, 카톡 갤러리 |
    | **Switch Triggers / Barriers** | **Trigger:** 주말 장보기 직전 “어디다 저장했지?” 하고 20분 넘게 못 찾은 경험**Barrier:** 노션/자동화 설정의 초기 세팅 부담 |
    | **Evidence** | “저장해둔 레시피를 찾지 못해서 결국 다른 메뉴를 해요…” |
    | **Priority (AOS / DOS)** | **AOS 3.0 / DOS 0.30 → Q1 핵심 혁신 기회** |
    | **Notes** | MVP 기능 적합도 최고: 자동 수집, 자동 정리, 태그링, 검색 |

    # 📌 **카드 2 — Adjacent User (효율 식단 플래너)**
    
    *(식단표·가족 관리·장보기 자동화가 강하게 필요한 층)*
    
    | 구분 | 내용 |
    | --- | --- |
    | **Persona / Segment** | **Adjacent – 효율 식단 플래너 (가족/다이어트/건강 중심)** |
    | **Situation** | 매주 월요일마다 **예산·칼로리·제철재료 기준으로 식단표를 짜야 하는 상황** |
    | **Job Statement** | 일주일 식단표를 자동으로 만들고, **장보기 리스트까지 자동 생성**되면 좋겠다 |
    | **Desired Outcome** | • 메뉴 고민 스트레스 제거• 장보기 준비 시간 **1시간 → 10분** 단축 |
    | **4 Forces** | **Push:** 매주 반복되는 식단표 스트레스와 메뉴 고갈**Pull:** 자동 플래닝+제철추천 기능에 대한 기대**Habit:** 네이버 메모/카톡/엑셀로 매주 반복 작성**Anxiety:** 가족 구성원 취향/알레르기 반영이 잘 될까? |
    | **Current Solutions** | 카톡 메모, 엑셀 수기 식단표, 유튜브 검색, 다이어트 앱 |
    | **Switch Triggers / Barriers** | **Trigger:** 갑작스런 가족 모임 또는 다이어트 시작으로 급히 식단 구성이 필요할 때**Barrier:** 자동추천이 실제 기호·예산에 맞지 않을까 하는 불안 |
    | **Evidence** | “매주 월요일만 되면 식단표 스트레스로 짜증이 나요.” |
    | **Priority (AOS / DOS)** | **AOS 1.6 / DOS 0.25 → Q2 확장 시장** |
    | **Notes** | ‘추천 알고리즘’·‘장보기 자동생성’ 추가 시 강력한 전환 예상 |

    # 📌 **카드 3 — Extreme User (앱 수집 요리러)**
    
    *(Pain 강도는 최고지만 온보딩 복잡 / TAM 상층)*
    
    | 구분 | 내용 |
    | --- | --- |
    | **Persona / Segment** | **Extreme – 앱 수집 요리러 (디지털 과부하층)** |
    | **Situation** | 레시피 앱 4개, 인스타 저장 수백개, 블로그 북마크 등 **툴 과다 사용으로 레시피가 ‘붕괴’** |
    | **Job Statement** | 이 난장판을 **한 번에 리셋하고**, 자동으로 정돈되는 “요리 지식 베이스”를 갖고 싶다 |
    | **Desired Outcome** | • 매번 검색 재작업 제거• 일관된 레시피 관리 경험 |
    | **4 Forces** | **Push:** 툴 난잡 → 매번 다시 검색해야 하는 고통**Pull:** “한 번에 리셋 + 자동 수집”이라는 메시지의 강력한 매력**Habit:** 앱 설치/해제 반복, 저장 습관 지속**Anxiety:** “또 새로 배우고 설정해야 해? 귀찮아 죽겠다.” |
    | **Current Solutions** | 앱수집 → 삭제 → 다시설치 패턴 반복 |
    | **Switch Triggers / Barriers** | **Trigger:** 스마트폰 초기화, 데이터 날림, 레시피 분실 경험**Barrier:** 설정 마찰, 온보딩 난이도, 기능 과다 인식 |
    | **Evidence** | “정리한다고 앱 다 깔아놓지만 결국 다시 검색부터 해요.” |
    | **Priority** | **AOS 4.0 / DOS 0.60 (Q3 고통 최대, 온보딩 난이도 높음)** |
    | **Notes** | 실험군으로 활용하면 데이터는 매우 유의미 |

    # 📌 **카드 4 — Non-user (아날로그 홈쿡)**
    
    *(TAM 하층 / 교육·입문 패키지 대상)*
    
    | 구분 | 내용 |
    | --- | --- |
    | **Persona / Segment** | **Non-user – 아날로그 홈쿡(디지털 입문층)** |
    | **Situation** | 카톡 사진·종이 레시피·요리책 위주로 조리 |
    | **Job Statement** | 누군가 대신 깔끔하게 세팅해주면 **레시피를 디지털로 한 번 정리해보고 싶음** |
    | **Desired Outcome** | • 요리를 더 쉽게 기억• 재사용 가능한 정리 시스템 확보 |
    | **4 Forces** | **Push:** 종이·사진 관리의 불편함**Pull:** 자동정리 시스템의 귀찮음 해결성**Habit:** 아날로그 방식 고착화**Anxiety:** 디지털화 자체에 대한 거부감 |
    | **Current Solutions** | 종이 요리책, 카톡첩, 냉장고에 붙여둔 쪽지 |
    | **Switch Triggers / Barriers** | **Trigger:** 이사/결혼/출산 등 생활 전환기**Barrier:** 디지털 진입장벽 |
    | **Evidence** | “불편하지만 지금도 그냥 써요.” |
    | **Priority** | **AOS 1.2 / DOS 0.0 (장기 교육 대상)** |
    | **Notes** | ‘입문 패키지’ 필요: 자동정리 + 미니 템플릿 제공 |
