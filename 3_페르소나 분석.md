1. 페르소나 스펙트럼 (레시피 OS 버전)
구분	페르소나 이름	한 줄 정의	대표 JTBD	위치
Core User	레시피 OS 빌더	Notion·시트·자동화로 “내 레시피 데이터베이스”를 이미 만들고 있는 사람	“인스타/유튜브/블로그 레시피를 한 군데로 모으고, 재료·칼로리·제철 태그까지 자동 정리하고 싶다.”	SOM (최초 타겟)
Adjacent User	효율 식단 플래너	건강·다이어트·가족식단 때문에 매주 식단표를 짜는 사람	“이번 주 예산·칼로리·제철재료를 기준으로 메뉴를 자동 추천받고, 장보기 리스트까지 한 번에 만들고 싶다.”	SAM (1차 확장)
Extreme User	앱 수집 요리러	각종 레시피 앱·인스타·블로그를 ‘다 깔아놓고’ 쓰다가 항상 정리가 안 되는 사람	“좋은 레시피는 다 저장하지만, 나중에 뭐가 어디 있는지 몰라서 또 검색한다. 이 지옥을 끝내고 싶다.”	TAM 상층 (재설계 필요)
Non-user	아날로그 홈쿡	손맛·종이 레시피·카톡 사진 위주로 요리하고, 자동화/미니멀리즘 개념엔 아직 거리감이 있는 사람	“지금도 불편하진 않지만, 누가 대신 세팅만 해준다면 써볼 의향은 있다.”	TAM 하층 (교육·온보딩 대상)

Core/Adjacent = 디지털 미니멀리즘에 적극적인 층 → 초반 세그먼테이션·MVP 검증 대상

Extreme = 문제는 가장 심하지만, UX/온보딩 난도가 높은 층

Non-user = 장기적으로 “디지털 입문/교육 패키지”로 데려올 수 있는 잠재 시장
